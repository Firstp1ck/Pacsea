pub fn send_query(app: &mut AppState, query_tx: &UnboundedSender<QueryInput>)
Expand description

Send the current query text over the search channel with a fresh id.

Side effects on app:

  • Increments and records next_query_id
  • Updates latest_query_id to the id sent

The id allows the receiver to tag results so the UI can discard any stale responses that arrive out of order.
